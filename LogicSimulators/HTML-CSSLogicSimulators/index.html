üß† CSS-Only Logic Gates & Latches

This project demonstrates basic logic gates and latches using only HTML and CSS.

üîå Core Logic

  • Inputs can be made with: <input type="checkbox"> <label for="id"><label>
  • Logic is implemented using CSS selectors:
    ":checked"- way to check input value
    ":not()" - acts as not gate
    "" - acts as and gate (write conditions joined without any space)
    "," - acts as or gate (write conditions separated by comma)
    "display:none | block" - can be used to toggle presence and absence of any element from the DOM
  • The state of one element (like an output) can be changed based on other elements (like inputs).

A T latch toggles its state when you turn it on, but holds the state when it's off.
But CSS can't store values‚Ä¶ or can it? ü§î

Here's the trick: since you're only allowed to toggle one button at a time, this setup creates a loop of four states: 

[0 ‚ü∂ 0] ‚Üí [1 ‚ü∂ 1]
   ‚Üë           ‚Üì
[1 ‚ü∂ 0] ‚Üê [0 ‚ü∂ 1]

The switch happens every second click. But a checkbox normally toggles on every click, so how do we fix that?

‚ñ∏ The solution is: every click swaps which button is visible.
‚ñ∏ Only one of those buttons (the first one) actually changes the output.
‚ñ∏ The other one is just a decoy to keep the toggle pattern going.

To complete the trick visually:
the button that controls the output is styled as if it's off,
and the other one is styled as if it's on.

Bonus insight: if you observe, the decoy (#t-b) acts as the the previous value of the output
and that is exactly how T latches work!

üíæ SR Latch

<div class="gate" title="SR Latch">
  <form class="gate-inputs">
    <input type="checkbox" id="sr-t" checked/> <label for="sr-t" title="Set (S)">S</label>
    <input type="checkbox" id="sr-s"/> <label for="sr-s" title="Set (S)">S</label>
    <input type="checkbox" id="sr-r"/> <label for="sr-r" title="Reset (R)">R</label>
    <button type="reset" class="reset-btn" title="Reset (R)">R</button>
  </form>
  <div class="gate-name">SR Latch</div>
  <div class="gate-outputs">
    <div class="output-light sr-output1"></div>
    <div class="output-light sr-output2"></div>
  </div>
</div>
/*
  SR - Latch  
                      [0, _] ["0", red] ["1", green] -> 0 (4th sate)
                        t   -     s     -     r               ‚Ü®
                    ‚Üô [1, _] ["0", gray] ["0", gray] -> 0 (0th sate)‚Ü∞
              (1st sate)                                            (3rd sate)
  ["1", green] [1, _] ["0", red] -> 1         ["0", red] [1, _] ["1", green] -> 0
       ‚Ü≥‚¨ë             (2nd sate) ["0", gray] [1, _] ["0", gray] -> 1    ‚¨è
*/
label[for="sr-t"],
.reset-btn,
#sr-r:checked + label[for="sr-r"],
#sr-s:checked + label[for="sr-s"] {
  display: none;
}

.gate:has(#sr-s:checked) label[for="sr-t"] {
  display: block;
}

.gate:has(#sr-t:checked):has(#sr-s:checked) label[for="sr-r"],
.gate:has(#sr-r:checked) input[type="checkbox"]:not(:checked) + label[for="sr-s"],
.gate:has(#sr-r:checked) input[type="checkbox"]:not(:checked) + label[for="sr-t"] {
  background: #f44336;
  color: white;
  box-shadow: inset 0 0 0 3px #c62828;
  pointer-events: none;
  opacity: 0.7;
  cursor: not-allowed;
}

.gate:has(#sr-r:checked) .reset-btn {
  all: unset;
  padding: 12px 0;
  border-radius: 6px;
  cursor: pointer;
  font-weight: 600;
  width: 48px;
  text-align: center;
  transition: all 0.25s;
  background: #4caf50;
  color: white;
  box-shadow: inset 0 0 0 3px #388e3c;
}

.gate:has(#sr-s:checked):not(:has(#sr-r:checked)) .sr-output1,
.gate:not(:has(#sr-s:checked):not(:has(#sr-r:checked))) .sr-output2 {
  background: #4caf50;
  box-shadow: var(--glow);
}

The SR latch has two inputs: S (Set) and R (Reset).
S stores the value, and can't take it back - only R can.

Using a similar strategy to the T latch (with a decoy input#sr-t), we apply the trick again ‚Äî but this time the decoy is active by default.
Here's how the state transitions map out: 

 "" ‚áí shown and clickable, '' ‚áí shown but not clickable
 T  S   R
[1 '0' "1" ‚ü∂ 0]
        ‚áÖ
[1 "0" "0" ‚ü∂ 0] ‚Üê ['0' 1 "1" ‚ü∂ 0]
     ‚Üì                     ‚Üë
["1" 1 '0' ‚ü∂ 1] ‚áÑ ["0" 1 "0" ‚ü∂ 1]
  • So the output is on if #sr-s is on but #sr-r is not.
  • when #sr-r goes from on to off it goes back to the start.
    To pull this off we hide the #sr-r and show the reset button and for this we need the inputs to be inside a form.
  • And to avoid the case where both buth inputs are on we use pointer-events: none;
